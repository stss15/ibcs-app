# A3.2 · Database design

### A3.2.1 · Describe database schemas
- Conceptual, logical, physical schemas and their abstraction levels.

### A3.2.2 · Construct ERDs
- Show entities, relationships, cardinality, modality; design efficient structures.

### A3.2.3 · Outline relational database data types
- Emphasise consistency and impact of incorrect choices.

### A3.2.4 · Construct relational tables
- Use keys (primary, foreign, composite, concatenated) to enforce integrity.

### A3.2.5 · Explain normal forms
- 1NF, 2NF, 3NF; terms: atomicity, unique identification, functional dependencies, partial-key, non-key/transitive dependencies.

### A3.2.6 · Construct a database normalised to 3NF
- Apply to scenarios such as libraries, hospitals, e-commerce, schools, inventory.

### A3.2.7 · Evaluate denormalisation
- Compare when denormalisation improves read performance versus duplication risks.
