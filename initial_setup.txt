IBCS APP — INITIAL SETUP (FLAT STACK POC)

Goal
Serve a React single-page app from GitHub Pages that talks directly to InstantDB (no middle tier) for teacher login, class creation, and student enrollment.

Prerequisites
- GitHub Pages enabled for the repo (`stss15/ibcs-app`)
- Node.js 18+ with npm
- InstantDB app ID and admin token (dev-only)

1. Project layout
```
frontend/               # React + Vite SPA
public/app-config.json  # APP_ID + admin token (dev only)
src/                    # Legacy static pages & styles (landing)
```

2. Install dependencies
```bash
npm install
```

3. Configure runtime (dev only)
`public/app-config.json`
```json
{
  "APP_ID": "fa61cd0c-d77e-44e5-919d-90a90ead7039",
  "INSTANT_ADMIN_TOKEN": "c88b98ee-4a55-4d83-ae15-57fdc04ed886"
}
```

4. Run locally
```bash
npm run dev --prefix frontend
```
Vite serves at `http://localhost:5173/ibcs-app/` (base already set).

5. Build
```bash
npm run build --prefix frontend
```
Publish `frontend/dist` to the branch used by GitHub Pages.

6. InstantDB collections (schemaless)
- teachers: `{ username, password, createdAt }
- classes: `{ className, description?, teacherUsername, createdAt, id }
- students: `{ name, username?, password, classId, teacherUsername, createdAt }

7. Demo bootstrap
The SPA seeds the demo teacher on first load if `teachers` lacks `username === "MrStewart"`.

8. Verification
1. Open https://stss15.github.io/ibcs-app/
2. Log in as Teacher (`MrStewart / Dragon-S25052`)
3. Create a class
4. Add a student (`Alice / student123`)
5. Refresh — class & student persist
6. Log in as `Alice` (student) to see assigned class

9. Cleanup after demo
- Rotate the InstantDB admin token
- Move to a server-side token minting flow
- Replace SHA-256 hashes with server-side bcrypt/argon2 in production
